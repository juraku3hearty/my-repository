# AIに設置してもらう手順（Claude / Gemini など）

このフォルダ一式を、コマンドを実行できるAI（Claude Code、Gemini CLI など
エージェント型AI）に渡せば、設置作業の大部分をAIが行えます。
人がやるのは「ログインの許可」と「APIキーの登録」だけです。

---

## 前提
- Node.js が入っていること
- ターミナルでコマンドを実行できるAI（Claude Code / Gemini CLI 等）
  ※ブラウザ版チャット（Claude.ai / Gemini アプリ）はコマンド実行ができないため、
  　「手順を教えてもらう」用途になります。

---

## AIにやってもらう作業（このフォルダで実行）

### 1. clasp（Google Apps Script のCLI）を用意
```
npx --yes @google/clasp --version
```

### 2. Googleにログイン  ← ここだけ人が「許可」を押す
```
npx --yes @google/clasp login
```
ブラウザが開くので、設置先のGoogleアカウントで「許可」する。

### 3. スプレッドシート付きのスクリプトを新規作成
```
rm -f .clasp.json
npx --yes @google/clasp create-script --type sheets --title "タイムカードOCR" --rootDir src
```
→ 確認シートになるスプレッドシートと、紐づくスクリプトが自動で作られます。

### 4. コードをアップロード
```
npx --yes @google/clasp push -f
```

---

## 人がやる仕上げ（2つだけ）

### A. 初期セットアップを実行
1. 作成された「タイムカードOCR」スプレッドシートを開く
2. メニュー「タイムカード読み取り → ① 初期セットアップ」をクリック
3. 初回は権限の承認 → 許可
   → 確認シートのタブ、未処理フォルダ、自動実行が自動でできます

### B. APIキーを登録
1. 拡張機能 → Apps Script → ⚙️ プロジェクトの設定 → スクリプトプロパティ
2. 名前：GEMINI_API_KEY ／ 値：自分のGemini APIキー（AIza…）
3. 保存 → メニュー「② 接続テスト」で「接続OK」が出れば完了

---

## これで完成
- 「タイムカード＿未処理」フォルダに写真（またはZIP）を入れる
  → 自動で読み取り → 確認シートに結果（怪しい所は色付き）
- 確認後、給与第一への転記は別紙のExcelマクロ（給与第一_転記マクロ.bas）を使用

## どのAIでも使えるか？
- Claude Code … 可（このまま実行）
- Gemini CLI（エージェント型） … 可（同じコマンドを実行できる）
- ブラウザ版チャット … コマンド実行不可（人が手順を見ながら打つ形）

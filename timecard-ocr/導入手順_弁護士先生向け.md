# タイムカード自動読み取りシステム ご利用案内

タイムカードの写真を **共有フォルダに入れるだけ** で、時刻を自動で読み取り、
裁判用ソフト「給与第一」に転記できる状態の表（確認シート）を作ります。

---

## 1. 何ができるか

```
① タイムカードの写真を Google ドライブの共有フォルダに入れる
        ↓（自動・10分ごと）AIが読み取り、怪しい所だけ色を付ける
② 「確認シート」で色の付いたセルだけ目視チェック・修正
        ↓
③「給与第一」へ転記（時刻の入力欄のみ・計算式は触りません）
```

- 写真は **手書き・印字どちらの打刻でも** 読み取れます。
- 二重・三重のチェックで誤りを炙り出します:
  1. **文字認識が得意なAIで2回読む**（同じ数字になるか）
  2. **さらに上位AIで全件を読み直し**、1のAIと食い違う所を検出
     （※同じAIだけだと同じ読み間違いで一致してしまい誤りが隠れるため、
       別の上位AIで全部突き合わせます）
  3. **論理チェック**：曜日の整合／出勤＜退勤／いつもの時間帯から大きく外れていないか
- 怪しいセルには色が付くので、事務の方は **色の付いた所だけ** 確認すれば済みます
  （全部を見比べる必要はありません）。色の付いたセルには上位AIの読み取り結果が
  メモ（コメント）で添えてあるので、判断の参考にできます。
  - 🟡 黄色 … AIが2回読んで数字が食い違った
  - 🟧 オレンジ … 曜日・時刻の矛盾、または上位AIと食い違った
  - 🟥 赤 … どうしても読み取れなかった（要入力）

## 2. 最初の準備（設置・最初の1回だけ）

> ここは少し技術的な作業です。事務所内で PC 操作に慣れた方が行ってください
> （30分ほど）。詰まったら構築担当にご連絡いただければサポートします。

### 2-1. Gemini API キーの取得

本システムは Google の AI「Gemini」で文字を読み取ります。利用に API キーが必要です。

1. ブラウザで **Google AI Studio**（https://aistudio.google.com/）を開く
   （事務所の Google アカウントでログイン）
2. 「**Get API key**（API キーを取得）」をクリック
3. 表示されたキー（`AIza...` で始まる文字列）をコピーして控える
4. **お支払い情報（クレジットカード）を登録**する（次項参照）

### 2-2. 料金について（お支払いのお願い）

- **API の利用には「お支払い情報（クレジットカード）の登録」が必要**です。
- ただし **実際にかかる費用はごくわずか** です。目安（あくまで概算・現在の料金は
  AI Studio の料金ページでご確認ください）:

  | 利用量 | 概算費用 |
  |---|---|
  | 1案件（タイムカード数十枚／2回読み＋上位AI再チェック込み） | **数十円程度** |
  | 1か月に数百枚読み取り | **数百円程度** |

- お支払い情報の登録は Google AI Studio / Google Cloud の課金設定から行えます。

> ご負担いただくのは **この Gemini の実費のみ** です。

### 2-3. システムの設置（確認シートに組み込む）

1. **確認シート**（タイムカードOCR）を開く
2. メニュー **拡張機能 → Apps Script** を開く
3. 開いたエディタの中身を消し、お渡しした **`Code.gs` の中身を全部貼り付け**
4. 先頭の設定（`CONFIG`）のうち、**`SHEET_NAME`** を確認シートの
   タブ名（画面下のタブの名前）に合わせる
5. 左の ⚙️ **プロジェクトの設定 → スクリプト プロパティ** で、
   - 名前：`GEMINI_API_KEY`　値：2-1 で控えたキー　を追加して保存
6. 確認シートを開き直すと、上部にメニュー **「タイムカード読み取り」** が出ます
   - **① 接続テスト** を実行 → 「接続OK」が出れば、キー・課金・モデルは正常です
     （初回は権限の承認画面が出るので許可。画像を使わないので費用はほぼゼロ）
   - **② 自動読み取りを設定** を実行 → **10分ごとの自動読み取り** が有効になります
7. **動作テスト**：写真を1枚だけ「タイムカード＿未処理」に入れ、メニュー
   「今すぐ読み取り」を押す → 確認シートに行が追加され、怪しい所に色が付けばOK
8. 以後は **写真を入れるだけ**（10分以内に自動処理されます）

## 3. 毎日の使い方（事務の方）

### 3-1. 写真を入れる

1. 共有フォルダ「**タイムカード＿未処理**」を開く
2. タイムカードの写真をアップロードする
   - ファイル名は自由です（カード上部の「令和○年○月分」をAIが自動で読みます）
   - 氏名でファイル名を付けておくと、確認シートにその氏名が入ります
     （例：`田中太郎.jpg`）
   - **ZIP（圧縮ファイル）のままでもOK** … 中の写真をまとめて自動で読み取ります

### 3-2. 自動で読み取られます

- **10分以内に自動で**読み取られ、確認シートに結果が追記されます。
- すぐ反映したい時は、確認シートのメニュー
  「**タイムカード読み取り → 今すぐ読み取り**」を押してください。

### 3-3. 確認シートでチェック

- **色の付いたセルだけ** 元の写真と見比べて確認・修正してください。
  - 🟡 黄色 … AIが2回読んで数字が食い違った
  - 🟧 オレンジ … 曜日・時刻の矛盾、または上位AIと食い違った
  - 🟥 赤 … 読み取れなかった（要入力）
- 色の付いたセルには **上位AIの読み取り結果がメモ** で添えてあります（参考に）。
- 色の無いセルはチェックを通過しているので、基本そのままでOKです。

---

## 4.「給与第一」への転記

### 4-1. 最初の1回だけの設定（計算規則シート）

1. **第１ 期間設定** … データの先頭の年月に合わせる
   （例：2024年12月〜のデータなら「**2024 年 12 月**」始まりに変更）
2. **第２ 賃金締め日** … 「**25**」に設定（このタイムカードは25日締めのため）

### 4-2. 転記

- 確認シートの内容を「給与第一」の **時間計算書** に転記します。
- 転記は時刻の **入力欄（始業・終業）だけ** に書き込み、**計算式には一切触れません**。
  そのため「給与第一」の計算が狂うことはありません。
  また、既に値が入っているセルは上書きしない安全設計です。

---

## 5. 困ったとき

- 色が多くて確認が大変 → 写真が傾いている/暗いと読み取り精度が落ちます。
  明るく・まっすぐ撮り直すと改善します。
- 設置がうまくいかない・キーやエラーが分からない → 構築担当までご連絡ください。

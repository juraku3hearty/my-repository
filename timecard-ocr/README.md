# タイムカードOCR（100チャレ No.2 / 弁護士・未払給料案件）

タイムカード画像を Gemini で **2モデル読み＋論理チェック** して確認シートに出力する GAS。
確認シートを人が目視 → VBA で「給与第一」へ転記、という流れの「入口〜確認」部分。

## 構成

```
Drive「タイムカード＿未処理」(画像投入)
        │  processUnprocessedTimecards()
        ▼
Gemini OCR ×2モデル（gemini-3.5-flash / gemini-2.0-flash）
        │  突き合わせ＋論理チェック
        ▼
確認シート（タイムカードOCR）… 怪しいセルに色付け
        │  ※ここは人が目視・修正
        ▼
VBA（給与第一.xlsm Module1）… 入力欄のみ・上書きなし
```

## セットアップ

1. **確認シートにスクリプトを設置**（コンテナバインド推奨）し、`Code.gs` を貼り付け。
2. **スクリプトプロパティ** に Gemini API キーを登録
   - `GEMINI_API_KEY` = `AIza...`（Google AI Studio で発行）
3. `CONFIG` を環境に合わせて調整
   - `SHEET_NAME` … 確認シートのシート名（既定 `シート1`）
   - `UNPROCESSED_FOLDER_ID` … `1VkjUeNaI4m4iwPMCZlm11uGhqBIMoSeE`（タイムカード＿未処理）
   - `CONFIRM_SHEET_ID` … `1aoLD-3WAQ8MoO1DMX_y49CObAKka_pxRcPQ_cOmV2l8`
   - `PROCESSED_FOLDER_ID` … 処理後に移動したい場合のみ設定
   - `CLOSING_DAY` … 締め日（このカードは `25`）
4. `processUnprocessedTimecards` を手動実行（初回は権限承認）。
   - 定期実行したい場合はトリガーを設定。

## 設計上の判断（重要）

- **2回とも同じモデルにしない**：同一モデルは同じ誤読を再現するため、一致＝正しいと
  誤認してエラーが隠れる。別モデル＋論理チェックで“見える化”する。
- **論理チェック**（一致していても発火）
  - 曜日：カード印字の曜日 vs 日付から計算した曜日 → 不一致は要確認（行ズレ検出）
  - 出 < 退 でなければ異常
  - 出勤 3:00–12:00 / 退勤 14:00–24:00 を外れたら要確認
- **締め日25・X案**：カード「N月分」の 26〜31日は前月、1〜25日は当月として
  カレンダー実日付で記録（`toCalendarDate_`）。
- **列マッピング A案**：出勤打刻→午前IN(E列) / 退勤打刻→午後OUT(H列)。
- 最終的な正しさは **確認シートの人手チェック** が担保（AIは「埋める＋怪しい所に色」まで）。

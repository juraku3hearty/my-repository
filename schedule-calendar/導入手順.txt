予定表 → Googleカレンダー 自動登録システム　導入手順


予定表（学校行事・部活など）の写真やPDFをフォルダに入れるだけで、
予定をAIが読み取り、確認用のシートに一覧化します。
内容を確認したあと、ボタンでGoogleカレンダーへ登録します。


────────────────────────────
１．全体の流れ
────────────────────────────

 (1) 予定表（写真/PDF）を Google ドライブの共有フォルダに入れる
 (2) 自動でAIが読み取り、確認シートに一覧化（怪しい所は色が付く）
 (3) 確認シートで、色の付いたセルだけ目視チェック・修正
 (4) ボタンを押すと、Googleカレンダーへ登録（時刻あり=時間付き / 無し=終日）

・学校の行事予定表、部活動の予定表など、形式が違っても読み取れます。
・同じ予定が二重に登録されないよう、重複は自動でスキップします。
・「OFF」「なし」など予定が無い日は登録しません。


────────────────────────────
２．最初の準備（設置・最初の1回だけ）
────────────────────────────

 ２－１．Gemini API キーの取得
　 (1) ブラウザで Google AI Studio（https://aistudio.google.com/）を開く
　 (2) 「Get API key（APIキーを取得）」をクリック
　 (3) 表示されたキー（AIza… で始まる文字列）を控える
　 (4) お支払い情報（クレジットカード）を登録する
　　　 ※API利用には登録が必要です。費用は1か月あたり数十〜数百円程度です。

 ２－２．スクリプトの設置
　 (1) 確認用のスプレッドシートを開く
　 (2) メニュー「拡張機能 → Apps Script」を開く
　 (3) 中身を消し、お渡しした Code.gs の中身を全部貼り付けて保存
　 (4) ⚙️ プロジェクトの設定 → スクリプトプロパティ で
　　　 名前：GEMINI_API_KEY　値：2-1で控えたキー　を追加して保存

 ２－３．初期セットアップを実行
　 (1) スプレッドシートを開き直すと、メニュー「予定表→カレンダー」が出ます
　 (2)「① 初期セットアップ」を実行（初回は権限の承認 → 許可）
　　　 → 未処理フォルダ・確認タブ・設定タブ・自動読み取りが自動で用意されます
　 (3)「② 接続テスト」で「接続OK」が出れば、キー・課金は正常です

 ２－４．お子さんの学年を入力
　 (1)「設定」タブを開く
　 (2) B1 のセルに学年（1〜3）を入力
　　　 ★毎年、進級したら この B1 の数字を変えるだけ★（コードは触りません）

 ２－５．登録先カレンダー（変更したい場合のみ）
　 ・初期設定では「普段お使いのメインカレンダー」に登録します。
　 ・専用カレンダーにしたい場合は、Code.gs の CALENDAR_ID を
　　 そのカレンダーのID に変更してください。


────────────────────────────
３．毎回の使い方
────────────────────────────

 ３－１．予定表を入れる
　 (1) 共有フォルダ「予定表＿未処理」を開く
　 (2) 予定表の写真またはPDFをアップロードする

 ３－２．自動で読み取られます
　 ・10分以内に自動で読み取られ、「確認」タブに一覧が出ます。
　 ・すぐ反映したい時は、メニュー「予定表→カレンダー → 予定表を読み取り」。

 ３－３．確認シートでチェック
　 ・色の付いたセルだけ、元の予定表と見比べて確認・修正してください。
　　　黄色　… AIが2回読んで食い違った
　　　オレンジ… 上位AIと食い違った／矛盾
　　　赤　　… 読み取れなかった（要入力）
　 ・色の付いたセルには上位AIの読み取り結果がメモで添えてあります。

 ３－４．カレンダーへ登録
　 ・メニュー「予定表→カレンダー → ★ カレンダーに登録」を押す
　 ・時刻があれば時間付き、無ければ終日で登録されます。
　 ・既に同じ予定がある日は、二重登録されません（自動スキップ）。
　 ・登録した行は「登録済」に印が付き、次回は重複しません。


────────────────────────────
４．困ったとき
────────────────────────────

・読み取りが多く外れる → 写真が傾いている/暗いと精度が落ちます。
　明るく・まっすぐ撮り直すと改善します。
・学年が変わった → 「設定」タブ B1 の数字を変えるだけです。
・設置やキーで分からない → 構築担当までご連絡ください。

データの保存先・外部送信については、別紙「セキュリティとデータの取り扱い」を
ご覧ください。読み取りのために画像はGoogleのAI（Gemini）へ送られますが、
有料APIのため学習には使われません。

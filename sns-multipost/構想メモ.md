# SNS多媒体・自動配信 サブスク構想メモ

> 既存の「インスタ＋Googleマップ自動更新（Make.com）」を、他店舗に**サブスク販売**するための構想。
> MEO業者さんと組んで横展開する前提。**この件はまた相談する。これは保管用メモ。**

最終更新：2026-06-24

---

## 1. やりたいこと

お店が**インスタに投稿するだけ**で、複数媒体に自動反映されるサービスを、月額サブスクで他店舗に売る。

```
お店：インスタに普通に投稿（いつも通り・昼1回など）
   ↓
MAYUの中央システム（Make.com）が「投稿あった？」と見に行く
   ↓ 新着あれば
  ├ 📍 Googleマップ（ビジネスプロフィール）
  ├ 💬 LINE公式アカウント
  ├ 📘 Facebook / 🧵 Threads
  ├ 🌐 WordPress（自社HP）
  └ 📧 メール配信（希望者）
   に自動で撒く
```

### なぜインスタを「入口（ソース）」にするか
- PC苦手・インスタしか触れない店主でも**今まで通り**で回る
- スプシ入力・写真URL化が**不要**（インスタに上げた時点で画像URLは自動でできる）

---

## 2. 解決した最重要論点：解約を効かせる

**問題**：今のまま（各店が自分のMakeで動かす）だと、契約が切れても使い続けられる。

**解決**：システムを**MAYUの中央**で動かす（各店に丸ごと渡さない）。
- お店ごとに「登録シート1行＋**有効/停止**」で管理
- 解約 → 停止にする → そのお店の投稿が**どこにも撒かれなくなる**＝確実に効く
- これは毎朝ニュースの「登録/配信列」と同じ仕組み
- **システムの所有はMAYU**。業者はリセラー（販売代理）。客が業者経由でも、止める権利はMAYU側

## 4. Make.com 費用・チェック頻度

- プラン：**Core $9/月（約¥1,400）**。無料はシナリオ2個で頭打ち＝多店舗は課金必須
- 課金で変わる主点：**シナリオ数2個→無制限**、クレジット1,000→10,000、間隔1分〜可
- クレジット消費の正体：**インスタを「見に行く」1回＝1クレジット（空振りでも）**。投稿回数ではなく**チェック回数**で決まる

### チェック頻度の設定（決定）
- **営業時間ベース：9〜20時、1時間おき（1日12回）**。朝の「今日の情報」用に朝だけ30分も可。深夜は止める
- お店が投稿したら**最大1時間で全媒体反映**＝¥3,000に見合う手厚さ
- **1店あたり原価 ≈ ¥76/月**（手厚くしてもタダ同然）
- 収容：$9プラン1本で**約20〜30店**。超えたら追加クレジット or Pro($16) or 間隔をゆるめる
- 重要：チェック頻度を上げて減るのは「1プランの収容数」だけ。**1店の利益（¥1,500−¥80）は不変**

## 5. 料金・役割分担

### 料金（2段＋将来3段目）
| プラン | 内容 | 月額 |
|---|---|---|
| ベーシック | 投稿を撒くだけ（マップ＋LINE公式） | ¥3,000 |
| フル | ＋FB/Threads/HP/メール配信 | ¥5,000 |
| （将来）おまかせ | ＋**AIが投稿案・キャプション生成**（後述） | ¥8,000〜 |

- **初期費用 ¥10,000 → MAYU総取り**（※条件：設置・連携作業はMAYUが全部やる、で筋が通る）
- **月額 折半**（MAYU / 業者）
- **Make代はMAYUの取り分から出す**（インフラ持ち）。業者の取り分は純利益＝やる気が出る

### 価格の考え方
- 相場（MEO代行¥1〜3万、SNS運用代行¥3〜10万）から見ると¥3,000は**安い**
- でも**集客・営業は業者**がやる＝MAYUは「技術提供者の取り分」でOK。原価¥80なので**高利益率**
- **投稿案は出さない＝労力ゼロ**だから安くて妥当。かつ「低労力×高利益×スケールする」SaaSの理想形
- 「安すぎ問題」は**フル¥5,000の受け皿**で解決。価格はあとで上げられる（まず安く広める）

### 役割
- **集客・営業・客との関係：業者（MEO業者）**
- **システム所有・設置・運用：MAYU**

## 6. 将来の上位プラン「おまかせ」＝ AIが投稿案まで作る

投稿の"中身"をAIが作る上位プラン。ネタ元は2つ：

### ① 写真から（Gemini Vision）※タイムカードOCRエンジン流用
```
店員：新入荷の商品（酒など）や手書きPOPを写真でパシャッ
   ↓
AI（Gemini Vision）
  ├ ラベルを読む（銘柄・産地・度数 等）
  └ 投稿文を生成「本日入荷！〇〇酒造の△△、すっきり辛口で…」
   → そのままインスタ投稿 → 全媒体に自動配信
```

### ② お天気から（天気API：無料）
```
その日の天気を取得（無料の天気API）
   ↓
AIが天気を絡めて投稿文を味付け
  例「今日は冷え込むので、熱燗に合う〇〇はいかが？」
    「猛暑日です。キンキンに冷えた△△が入荷しました」
```
- 天気は無料API（気象庁/Open-Meteo等）で取れる＝原価ほぼ0
- ①写真 と ②天気 は組み合わせOK（写真で銘柄、天気で切り口）

- 「写真撮るだけ／何もせず天気連動で、投稿文まで完成」＝¥8,000〜の価値
- ゼロから作らない（timecard-ocr のOCRエンジン＋毎朝ニュースのGemini活用が土台）

# WordPress サクサク投稿の仕組み(スプレッドシート → WordPress)

リポジトリの `wp_poster.js` をGASに追加すると、スプレッドシートに書いた記事を
WordPressへ一括投稿できる。記事はClaudeに量産させてシートに貼る運用を想定。

## セットアップ(初回のみ・10分)

1. **WordPress側**: 管理画面 > ユーザー > プロフィール > 「アプリケーションパスワード」で
   新規発行(名前は「GAS投稿」など)。表示されたパスワードをコピー
2. **GAS側**: このリポジトリを `clasp push` するか、`wp_poster.js` の中身を
   GASエディタに貼り付け
3. GASエディタ > プロジェクトの設定 > **スクリプト プロパティ** に登録:
   - `WP_URL` = `https://サイトのURL`(末尾スラッシュなし)
   - `WP_USER` = WordPressのユーザー名
   - `WP_APP_PASS` = 発行したアプリケーションパスワード
4. スプレッドシートを開いた状態で `setupPostSheet()` を実行 → 「投稿キュー」シート完成

## 日々の運用

1. 「投稿キュー」シートに記事を書く(タイトル/本文HTML/カテゴリ/タグ/アイキャッチURL)
2. ステータスを「投稿する」にする
3. `postQueued()` を実行 → 投稿され、URL・日時がシートに書き戻される
   - 毎時の自動投稿にしたい場合は `createPostTrigger()` を一度実行
   - カテゴリ・タグは名前で書けばOK(WordPress側に無ければ自動作成)
   - 失敗した行はステータス「エラー」+メモ欄に理由が入る

## 記事量産の流れ(想定)

Claudeに「候補店舗リストの◯◯の紹介記事を書いて」と頼む
→ 出てきたHTMLをシートに貼る → ステータス「投稿する」→ 実行。これだけ。

## サブディレクトリ構成(決定版の提案)

**方針: 地域で丸ごと分ける(2026-06-10 決定)。** エリアを増やすときは同じ型を複製する。

```
サイトURL/
└── chitose/             ← 千歳のまとめトップ(消費者向けハブ。新規オープン上位表示枠もここ)
    ├── shop/◯◯◯/      ← 店舗ホームページ(例: /chitose/shop/salon-lien/)
    ├── open/            ← 新規オープン情報
    ├── event/           ← イベント情報
    ├── jobs/            ← 求人・募集系
    ├── column/          ← お店紹介・コラム記事
    └── partner/         ← 掲載店向け営業LP(「千歳の」と語りかけるエリア密着版)

恵庭に展開したら /eniwa/ 以下に同じ構成を複製
```

命名メモ:
- 営業LPに /recruit・/boshu は将来の /jobs(求人)と混同するため避ける
- /chitose/ 直下(一等地)は消費者向けまとめトップに使う。営業LPは /chitose/partner/
- 店が稀に隣の市へ移転した場合はURLが変わる(旧URLからリダイレクトを張れば実害なし)

ルール:
- **スラッグは短い英小文字**。日本語URLは共有時に%エンコードで崩れるのでNG
- **「main」のような意味のない名前は使わない**(後から変えられない)
- **店舗URLにエリアや業種は入れない**(/shop/店名/ で固定)。
  恵庭・苫小牧に広げても、店が業態変更してもURLが変わらないため。
  エリア・業種はWordPressのカテゴリ/タグで持たせて絞り込みページを作る
- 「千歳 美容室」等の検索順位はURLよりタイトル・本文・内部リンクで決まるので、
  URLをシンプルに保ってもSEO上の損はない

店舗ホームページの置き方は2通り(どちらでも可):
- A. WordPressの固定ページ(親ページ=shop)として作る
- B. 今回の静的HTML雛形をサーバーの /shop/店名/index.html にアップする
  (雛形のデザインをそのまま使えるのでまずはBが楽)

# 🎬 広告動画工房 — 整骨院 動画広告 制作・管理システム

90〜120秒の縦型動画広告を「台本生成 → スタッフ声ナレーション → AI動画 → 合成 → AB計測」まで一気通貫で回す、うち専用システム。

## 全体像

```
┌─ Googleスプレッドシート(司令塔・DB) ─────────────────┐
│ 素材ライブラリ / 台本 / ジョブ / バリアント / AB結果 / レポート │
└──────┬───────────────────────────┬──────┘
       │ GAS(メニュー操作)              │ ポーリング(60秒ごと)
       ▼                              ▼
  Gemini で台本3案生成           VPSワーカー(Node.js)
                                  ├ Fish Audio: スタッフ声ナレーション
                                  ├ Higgsfield: AI動画生成(差し替え可)
                                  ├ ffmpeg: 撮影素材+AI動画+音声を合成
                                  └ 完成mp4をDriveへ → シートにURL・コスト記入
```

- **管理は全部スプレッドシート。** 何がいくらで作られて、どの媒体でどれだけ予約が取れたか全部残る
- **プロバイダはアダプタ方式。** Fish Audio / Higgsfield が気に入らなくなったらアダプタ1ファイル追加+環境変数変更で乗り換え
- **90〜120秒特化。** ナレーションの長さ=動画の長さ。素材が足りなければ自動ループで尺を埋める

## 日々の使い方(セットアップ後)

1. スプレッドシートを開く → メニュー「🎬 広告動画工房」
2. **① 台本を3パターン生成** — カテゴリ(脱毛/整体/睡眠)とターゲットを入れると LLM が A(共感)/B(信頼)/C(オファー) の3案を書く
   - カテゴリごとの「サービス内容・強み・NG表現」(事業カテゴリシート)が自動でプロンプトに入るので、脱毛広告に「永久脱毛」等の地雷表現が混ざらない
   - **ボイス一覧シートにスタッフの声を登録しておくと、AIがターゲットに合う声を理由つきで推奨**(例: 男性向け脱毛→男性スタッフの声)。ジョブ発行時に自動でその声が使われる
   - LLMはデフォルトGemini(無料)。勝負どころは設定シート「台本LLM」を `claude` に変えると Claude(claude-opus-4-8、台本1回あたり数円〜十数円)で高品質生成。要スクリプトプロパティ `ANTHROPIC_API_KEY`
3. **② 動画ジョブを発行** — 台本ID+使う素材ID+AI動画プロンプト+**配信店舗**を指定
   - 店舗をカンマ区切りで複数指定すると、**本体は共通のままエンドカード(店舗情報クリップ)だけ差し替えた動画が店舗の数だけ自動生成**される
   - 事前に各店舗の店舗情報クリップ(住所・電話・外観など5秒程度)を素材登録し、「店舗一覧」シートで紐付けておく
4. あとは待つだけ。ワーカーが処理して「ジョブ」シートに 出力URL と 作成コスト(円) が入る(LINE通知も可)
5. 広告を配信したら **③ AB結果を入力**(表示回数・クリック・予約数・広告費)
6. **④ レポート更新** — バリアント×媒体ごとに CTR / CVR / **総コスト(広告費+作成費)** / **CPA(予約1件あたり円)** を集計、一番安く予約が取れたものに「★勝ち」

## 初期セットアップ

### A. スプレッドシート+GAS(15分)

1. 新規スプレッドシートを作成、URLからIDを控える
2. 拡張機能 → Apps Script → `gas/` の6ファイルを貼り付け
3. スクリプトプロパティを設定:
   - `SPREADSHEET_ID` … 手順1のID
   - `GEMINI_API_KEY` … [Google AI Studio](https://aistudio.google.com/) で取得(無料枠)
4. `initialize()` を手動実行(シート7枚が自動生成される)
5. 設定シートの「院名」を自院名に変更

### B. Fish Audio 声クローン(20分)

1. [fish.audio](https://fish.audio/) でアカウント作成 → APIキー取得
2. スタッフの声を録音(雑音のない環境で1〜3分、自然な話し方で)。**男女両方いると理想**(男性向け脱毛広告などでAIが使い分ける)
3. Fish Audio でボイスモデル作成 → **モデルID(reference_id)** を控える(スタッフごとに1モデル)
4. スプレッドシートの「ボイス一覧」シートに登録(ボイスID・名前・性別・声の印象)。1人だけなら設定シート「デフォルトボイスID」に貼るだけでもOK
5. 声の使い分けの優先順位: 台本のAI推奨ボイス > 事業カテゴリのボイスID > デフォルトボイスID

### C. Higgsfield(10分)— 2つの使い方

**C-1. 公式MCP(おすすめ・まずこっち)**

Higgsfield 公式のMCPサーバー(`https://mcp.higgsfield.ai/mcp`)を Claude に繋ぐと、会話しながらAI動画クリップを生成できる。APIキー不要、Higgsfieldアカウントで認証、プランのクレジットがそのまま使える(無料枠は月150クレジット)。

- Claude Code: このリポジトリ直下の `.mcp.json` に登録済み。初回にブラウザ認証するだけ
- Claude(Web/デスクトップ): 設定 → コネクタ → カスタムコネクタ追加 → URL に `https://mcp.higgsfield.ai/mcp`

運用: Claudeに「施術ベッドで骨盤矯正している明るい院内の5秒クリップを作って」と頼む → 生成された動画をDriveに保存 → 素材ライブラリに登録 → ジョブ発行。ワーカー側は `.env` で `VIDEO_PROVIDER=manual` にしておく(合成のみ担当)。

**C-2. Cloud API(完全自動化したくなったら)**

[Higgsfield Cloud](https://cloud.higgsfield.ai/) で APIキー/シークレットを取得して `.env` に設定、`VIDEO_PROVIDER=higgsfield`。ジョブ発行だけで動画生成まで全自動になる。エンドポイントが変わっていたら `worker/src/providers/video/higgsfield.js` の `ENDPOINTS` を公式ドキュメントに合わせて修正。

### D. VPSワーカー(30分)

```bash
# ConoHa VPS (163.44.114.69) 上で
sudo apt install -y ffmpeg
mkdir -p /root/ad-studio && cd /root/ad-studio
# このリポジトリの ad-studio/worker/ を配置
npm install
cp .env.example .env && nano .env   # 各キーを記入
```

**Googleサービスアカウント**(ワーカーがシートを読み書きするため):

1. [Google Cloud Console](https://console.cloud.google.com/) → プロジェクト作成 → Sheets API と Drive API を有効化
2. サービスアカウント作成 → 鍵(JSON)をダウンロード → `/root/ad-studio/service-account.json` に配置
3. スプレッドシートと出力用Driveフォルダを、サービスアカウントのメールアドレスに**編集者**で共有
4. 出力用DriveフォルダのIDを `.env` の `DRIVE_OUTPUT_FOLDER_ID` に

**起動:**

```bash
node src/index.js --once   # 動作確認(1ジョブだけ処理)
tmux new -s ad-studio 'npm start'   # 常駐(他サービスと同じtmux運用)
```

### E. 素材登録(撮影のたび)

1. 施術風景などをスマホで撮影(**縦向き推奨**。横でも自動で縦にクロップされる)
2. Google Drive にアップ → ファイルIDを控える
3. 「素材ライブラリ」シートに1行追加(素材ID は `MAT-001` など自由、DriveファイルID・種類・タグを記入)

## シート構成

| シート | 役割 |
|---|---|
| 事業カテゴリ | 脱毛/整体/睡眠 など事業ごとのサービス内容・強み・**NG表現**の台帳。台本AIが参照 |
| 店舗一覧 | 店舗ごとのエンドカード素材の紐付け。店舗別動画の自動生成に使用 |
| 店舗比較 | 自動集計。**内容(カテゴリ)ごとに店舗間のCTR/予約/CPAの差**を見る |
| ボイス一覧 | Fish Audioの声モデル台帳(スタッフ名・性別・声の印象)。AIが台本ごとに最適な声を推奨 |
| 素材ライブラリ | 撮影した動画クリップの台帳(Drive ID・種類・タグ) |
| 台本 | Gemini生成の台本ストック(フック/本文/CTA) |
| ジョブ | 制作キュー。状態・出力URL・**作成コスト(円)**・エラーが残る |
| バリアント | 広告1本=1行。どの台本×素材の組み合わせかの台帳 |
| AB結果 | 日々の配信実績(表示・クリック・予約・広告費) |
| ABレポート | 自動集計。CPA順で「★勝ち」判定 |
| 設定 | 院名・デフォルトボイスID・尺・アスペクト比 |

## トラブル時

- **ジョブが error になった** → ジョブシートのエラー列に原因が入る。直したらメニュー「エラージョブを再実行」
- **Higgsfield のAPIが変わった** → `worker/src/providers/video/higgsfield.js` の `ENDPOINTS` を公式ドキュメントに合わせて修正
- **別の動画AIに乗り換えたい** → `worker/src/providers/video/` に新アダプタを作り、`.env` の `VIDEO_PROVIDER` を変更(詳細は `docs/ARCHITECTURE.md`)
